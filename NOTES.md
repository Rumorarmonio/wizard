# Рабочие заметки проекта

## Проект

Frontend-приложение onboarding wizard без бэкенда на Nuxt 4, Vue 3 и TypeScript.
Основной сценарий: пользователь проходит последовательные шаги формы, может двигаться вперёд и назад, сохраняет прогресс локально, проверяет введённые данные на шаге Review и завершает сценарий на экране Complete.

## Цель

Собрать минимальную рабочую версию многошагового мастера onboarding с шагами Account, Profile, Preferences, Review и Complete, с валидацией по шагам, сохранением данных в `localStorage` и аккуратным минималистичным интерфейсом без лишних библиотек.

## Текущее состояние

Инициализирован минимальный Nuxt 4 проект с TypeScript и базовой директорной структурой.
В проекте уже подключён глобальный слой стилей и подготовлены директории под страницы, компоненты, feature-модули, composables, вспомогательные утилиты и типы.
Добавлена базовая инфраструктура кодстайла: `ESLint`, `Prettier` и `Stylelint`, адаптированная под `Nuxt 4` и отдельные `SCSS`-файлы.
Подготовлена доменная модель wizard, порядок шагов и клиентское сохранение черновика в `localStorage`.
Реализованы UI и пошаговая валидация для шагов `Account`, `Profile` и `Preferences`.
Реализованы шаг `Review` со сводкой введённых данных и экран `Complete` после подтверждения.
Собран минимальный UI с отдельным модульным `SCSS` и локальным fallback-состоянием для динамических частей wizard на время восстановления черновика после перезагрузки.
Подготовлена инфраструктура для автодеплоя на `GitHub Pages` через `GitHub Actions` со статической генерацией Nuxt.

## Ключевые решения

- Стек проекта: Nuxt 4 + Vue 3 + TypeScript.
- Стили: модульные SCSS-файлы, вынесенные в отдельные файлы рядом с компонентами или страницами.
- Глобальные стили: отдельный инфраструктурный слой в `app/assets/styles`, подключённый через `nuxt.config.ts`.
- Архитектурный фокус: только frontend, без бэкенда и без внешней UI-библиотеки.
- Основной UX-поток: последовательные шаги с кнопками `Next` / `Back` и запретом перехода вперёд при невалидных данных текущего шага.
- Сохранение прогресса: черновик состояния wizard хранится в `localStorage`, чтобы пользователь мог продолжить после перезагрузки.
- Финальный поток: перед завершением пользователь видит шаг Review со сводкой всех введённых данных, после подтверждения попадает на экран Complete.
- Выбран базовый формат состояния: единый `useWizardState` composable поверх общей структуры draft с шагом, данными формы, списком завершённых шагов и метаданными сохранения.
- Порядок шагов зафиксирован как `Account -> Profile -> Preferences -> Review -> Complete`.
- Сохранение черновика реализуется напрямую через `localStorage` с безопасной нормализацией загруженных данных и без внешних зависимостей.
- Для этапа моделирования приняты минимальные поля:
  `Account` — `email`, `password`;
  `Profile` — `firstName`, `lastName`, `jobTitle`, `location`;
  `Preferences` — `interests`, `notifications`, `workspaceMode`.
- Для этапа реализации шагов выбрана простая встроенная валидация без внешних библиотек: чистые функции проверки и ошибки по текущему шагу.
- Wizard на текущем этапе собран на одной странице как композиция шагов поверх общего composable состояния.
- Шаг `Review` собирает summary из уже сохранённых данных без отдельного стор-слоя и без дублирования формы.
- После подтверждения на `Review` flow переводится на `Complete`, а сброс сценария выполняется через уже существующий `resetDraft()`.
- Для сценария reload SSR сохранён для shell-структуры страницы, а fallback показывается только внутри динамических блоков sidebar и panel до завершения гидрации.
- Для деплоя на `GitHub Pages` приложение собирается как статический `SPA`: в `nuxt.config.ts` включён `ssr: false`, а `baseURL` задаётся через переменную окружения `NUXT_APP_BASE_URL` в workflow.
