# Инструкции для агента

## Общие правила работы

- Работай пошагово.
- Не реализуй несколько крупных этапов сразу без явного запроса.
- Перед большими изменениями сначала предложи краткий план.
- Изменяй только необходимые для текущего шага файлы.
- Не делай лишних рефакторингов и не изменяй несвязанные части проекта без запроса.
- Если задача касается OpenAI API, Codex, MCP, Agents SDK или другой документации OpenAI, используй MCP-сервер `openaiDeveloperDocs`.
- Если задача касается общей документации библиотек, фреймворков, конфигурации или интеграций, используй MCP-сервер `context7`.

## Ограничения

- Не меняй настройки ESLint, Prettier, TypeScript, Vite и другие конфиги без необходимости.
- Если для задачи требуется изменить конфиги или структуру проекта, сначала объясни зачем.
- Не добавляй новые зависимости без явной необходимости.
- Не откатывай и не переписывай вручную внесённые изменения пользователя без явного запроса.

## Кодстайл

- Используй TypeScript.
- Предпочитай понятные имена переменных и функций.
- Не усложняй архитектуру раньше времени.
- Держи код простым и читаемым.
- Добавляй комментарии только там, где нужно объяснить причину, а не очевидные действия.
- По возможности предпочитай простые функции и понятные структуры данных вместо преждевременного усложнения.

## Формат работы

- После изменений перечисляй изменённые файлы.
- Кратко объясняй, что было сделано.
- Если есть сомнения в архитектурном решении, сначала предложи варианты, а не меняй код сразу.
- Если задача сформулирована неоднозначно, сначала кратко обозначь допущения, на которых основано решение.

## Работа с заметками

- Используй `NOTES.md` как основной файл с рабочими заметками по проекту.
- Используй `PLAN.md` как основной файл плана реализации.
- В `NOTES.md` фиксируй контекст проекта, важные решения, уже выполненные шаги, открытые вопросы и ближайший следующий шаг.
- В `PLAN.md` фиксируй цель текущего плана, этапы реализации, статус этапов и следующий крупный этап.
- Не перезаписывай `NOTES.md` и `PLAN.md` целиком без необходимости.
- Обновляй только релевантные разделы, сохраняя уже зафиксированный полезный контекст.
- Обновляй `NOTES.md` после завершения значимого этапа работы, если изменились контекст проекта, решения, выполненные шаги или открытые вопросы.
- Обновляй `PLAN.md`, если изменились этапы реализации, статус этапов или следующий крупный этап.
- Не обновляй `NOTES.md` и `PLAN.md` для мелких локальных фиксов, если они не меняют контекст проекта, план или текущее состояние этапов.

## Проверка результата

- После реализации шага указывай, что именно стоит проверить вручную.
- Если код не был проверен запуском, явно сообщай об этом.
- Если были изменения в файлах проекта, то в конце ответа предлагай краткое сообщение для коммита на русском и английском. Если не было, то не предлагай сообщение.
- На английском пиши в стандартном commit-стиле, в повелительном наклонении.
- На русском пиши в третьем лице, в стиле "Добавляет", "Исправляет", "Удаляет".
